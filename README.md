# RoboMission Junior 得点計算

現在のアプリバージョン：`v1.6.0`

WRO 2026 RoboMission Juniorの、判定写真付き得点計算・記録Webアプリです。スマートフォン、タブレット、PCで利用でき、PWAとしてインストールすると採点と全判定写真をオフラインでも利用できます。ルールPDFは必要になった時だけ読み込みます。

# 公開 URL
# https://ecleaire.github.io/ROBOMISSION_POINTToul/#/score  

## 主な機能

- 6ミッション、最大230点の得点計算
- チェックなしを0点として扱うスプレッドシート型の入力
- オブジェクトごとの全判定例を一枚の比較表で確認できる判定写真モーダル
- 全30枚の判定写真をPWA初回準備時に保存し、未閲覧でもオフライン表示
- 途中状態の端末内保存（`localStorage`）
- APIキー別の端末内保存と記録一覧
- 再描画中も入力内容を保持するアカウント切替フォーム（Enter送信対応）
- 練習した回ごとのメモ一覧・新規作成・編集
- 未送信の採点結果・動画を端末に退避し、オンライン復帰時に自動再送
- 通信状態、端末保存、未送信件数を採点画面で常時確認
- 練習記録の平均・最高・前回比・得点推移・ミッション別比較
- 動画・写真・メモの有無による履歴絞り込みと、記録カード内のミッション別得点
- 採点画面の文章メモへコート画像を書き込み、得点と一緒に保存
- 練習回に紐づかないアカウント別メモの作成・編集・削除
- メモ1件につき最大5枚をアプリ内の背面カメラで撮影し、写真ごとに図形・ペン・文字を書き込み
- メモ写真は端末で最大1600px・JPEGへ軽量化し、非公開Google Driveへ保存
- メモを画面へ即時反映し、一覧の再取得なしでバックグラウンド保存
- メモと25MB以下の動画を採点結果と一緒に保存
- 管理画面からのアカウント追加、チーム名変更、APIキー再設定
- 管理キーによる全アカウント記録の閲覧
- 管理アカウントから保存先チームを選んで採点・動画・メモを保存
- 日付・期間・得点範囲・キーワード検索・並び替え
- 練習記録を開いた時・アプリへ戻った時・表示中60秒ごとに自動更新
- Google Apps Script経由で、キー別のシートへ競技時間・得点・メモを追記
- 得点を先に保存し、動画はバックグラウンド送信する高速保存
- 再生した記録動画を端末へ最大3件キャッシュし、2回目以降は通信なしで高速再生
- アプリで削除した練習記録を一覧から隠し、スプレッドシートには「削除」チェック付きの赤い行として保管
- Google翻訳版ルールPDFのアプリ内表示・文字検索・全画面表示（iPadは複数ページ対応ビューアへ自動切り替え）
- Google翻訳版と兵庫予選会の補足・ローカルルールPDFを、ルール画面内のボタンで切り替えて表示
- Google翻訳版-General-Rulesを含む3種類のPDF切替

- 兵庫予選会ロボミッション（エキスパート競技）の日程・会場・Googleマップ
- 大会会場の遅延読み込みミニマップと「開催概要 詳細」リンク

## 使い方

1. 練習キー `A`、`B`、`C` のいずれかを入力します。
2. 動画を残す場合は、ストップウォッチ直下の「録画開始」を押します。背面カメラの録画中も競技ストップウォッチを独立して操作でき、「録画停止」で動画を確定します。動画が不要な場合は何も押しません。
3. 表上部のストップウォッチを開始します。終了すると競技時間へ自動反映され、表の一番下で手動修正もできます。
4. 表のチェック欄から各対象の得点を選びます。チェックなしは0点で、同じ欄をもう一度押すと解除できます。
   汚れは、石畳に触れていない個数を0～10から選択します。
5. 「結果を見る」を押し、遺物の色と合計を確認します。
6. 既存動画を使う場合は、表の一番下にある「動画を選択」を使用します。メモと動画は任意です。
7. Googleスプレッドシートへ記録します。得点保存後はすぐ採点画面へ戻り、動画はバックグラウンドで送信されます。「動画の保存が完了しました」と表示されるまではアプリを閉じないでください。
8. 上部メニューの「練習記録」から、現在のキーに対応する記録だけを確認できます。
   「メモ」では各練習回のメモを新しく作成したり、既存のメモを編集したりできます。
9. 不要な記録は「この記録を削除」で一覧から消せます。元の行はスプレッドシートに赤い行で残り、「削除」のチェックを外すとアプリへ再表示できます。完全に消す場合は、スプレッドシートでその行自体を削除します。
10. 上部メニューの「リンク・大会情報」から兵庫予選会の日程・会場、WROホームページ、ルール、Q&A、関連動画、公開URL QRコードを確認できます。

## 開発

Node.js 20以上を使用します。

```bash
npm install
npm run dev
```

テストと本番ビルド：

```bash
npm test
npm run build
```

ビルド結果は`dist/`に生成されます。GitHub Pagesのリポジトリ名が変わる場合は、`vite.config.ts`の`base`も変更してください。

## Googleスプレッドシート連携

送信先スプレッドシートIDは`gas/Code.gs`に設定済みです。APIキーはコードへ直接書かず、Apps Scriptの「プロジェクトの設定」→「スクリプト プロパティ」で次の3項目を設定します。

| プロパティ | 値 |
| --- | --- |
| `API_KEY_A` | `A` |
| `API_KEY_B` | `B` |
| `API_KEY_C` | `C` |

各プロパティの値に一致したキーだけが受け付けられ、同じスプレッドシート内の `練習記録_A`、`練習記録_B`、`練習記録_C` に分かれて保存されます。既存3アカウントのチーム名は、管理画面で変更すると `ACCOUNT_NAME_A`、`ACCOUNT_NAME_B`、`ACCOUNT_NAME_C` に保存されます。

管理画面では、既存アカウントのチーム名・APIキー変更と、新しいアカウントの追加ができます。追加アカウントはスクリプト プロパティ `ACCOUNT_CONFIG_JSON` に保存され、チーム名を含まないランダムな内部IDと `練習記録_ACC_...` シートを使用します。既存APIキーの値は管理画面へ返さず、変更時に新しいキーを入力する方式です。

管理者パスワードは、名前が `MASTER_PASS` で始まるスクリプト プロパティ（例：`MASTER_PASS`、`MASTER_PASS_2`）へ設定します。複数設定した場合はいずれの値でも管理画面へログインできます。値の大文字・小文字は区別しません。「ルール」または「リンク」を10回押すとモードメニューに「管理」が表示され、管理画面では全アカウントの記録・アカウント設定を管理できます。

APIキーも値の大文字・小文字を区別しません。通常のAPIキー入力欄へ管理者パスワードを入力した場合は、管理画面へ直接移動します。通常アカウントのAPIキーを端末へ保存するかはログイン時に選択でき、管理者パスワードは端末へ保存しません。

### チーム名のプライバシー

- チーム名と追加アカウントのAPIキーはGASのスクリプトプロパティだけに保存します。
- チーム名・APIキーを公開リポジトリ、JavaScript、URL、シート名へ埋め込みません。
- 通常アカウントには認証した自分のチーム名だけを返し、全チーム名は管理者認証時だけ返します。
- 既存APIキーは管理画面にも表示しません。APIキー変更時は新しい値で上書きします。

### 動画の保存とプライバシー

- 動画は25MB以下に制限し、GAS実行ユーザーのGoogle Drive内に作られる非公開フォルダへ保存します。アプリ内録画は720p・約24fps・1Mbps・音声なし・最長3分を目安に容量を抑えます。
- スプレッドシートには非公開の内部ファイルIDだけを保存し、アプリの記録一覧へDrive URLやファイルIDを返しません。
- 動画の取得時にもAPIキーを検証し、保存したアカウント本人または管理者だけに動画データを返します。
- 初回保存時にスクリプト プロパティ `VIDEO_FOLDER_ID` が自動作成されます。値を公開したり手動共有したりしないでください。
- 動画はオフライン保存の対象外です。閲覧にはGASとGoogle Driveへの通信が必要です。
- 認証付き操作はPOSTのみとし、GETのクエリ文字列による認証は受け付けません。

Apps Scriptプロジェクト（スクリプトID `1dZLK_-vhhS-uTBt8huiPr34bBiMobmk723eGAYUALNFtPOu5vIRVeLWo`）を再デプロイする場合は次を行います。

1. `gas/Code.gs`の内容をApps Scriptエディタへ貼り付けて保存します。
2. 「プロジェクトの設定」→「スクリプト プロパティ」で `API_KEY_A`、`API_KEY_B`、`API_KEY_C` と、`MASTER_PASS` から始まる管理者パスワードを設定します。
3. 「デプロイ」→「新しいデプロイ」→種類「ウェブアプリ」を選びます。
4. 実行ユーザーを自分、アクセスできるユーザーを利用環境に合う公開範囲にします。選手がログインせず使う場合は「全員」を選びます。
5. 初回承認後、発行された`https://script.google.com/macros/s/.../exec` URLをコピーします。
6. URLを`src/config.ts`の`DEFAULT_GAS_WEB_APP_URL`へ設定し、アプリを再ビルドします。

現在のデプロイURLはアプリに設定済みです。結果画面の「この結果を記録する」を押すと、選択中のキーに対応するシートへ1採点1行で追加されます。

> `A`・`B`・`C` は練習グループを簡単に切り替えるためのキーで、パスワードのような強い認証ではありません。公開URLとキーを知っている人は、そのキーの記録を閲覧・追加できます。
